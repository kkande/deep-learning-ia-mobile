1. Travail réalisé durant le Sprint 1

1.1 Mise en place de l’environnement de développement

Les activités suivantes ont été réalisées pour initialiser le projet : * Création du projet Android sous Android Studio Hedgehog. * Configuration de Gradle avec les dépendances TensorFlow Lite, CameraX et ML Kit (Version Catalog libs.versions.toml). * Mise en place de l’architecture MVVM / Clean Architecture (couches data, domain, presentation). * Initialisation du dépôt Git et configuration des branches (main, develop, features/*). * Intégration de Jetpack Compose pour l’interface utilisateur.

1.2 Intégration du modèle de reconnaissance d’émotions

Le modèle TFLite de détection d’émotions a été intégré avec les étapes suivantes : * Sélection et conversion du modèle MobileNetV2 quantifié (INT8) en format .tflite. * Ajout du fichier modèle dans le répertoire assets/ du projet Android. * Création du TFLite Interpreter avec délégation GPU activée pour optimiser les performances. * Implémentation du pipeline d’inférence : capture → prétraitement (redimensionnement, normalisation) → inférence → post-traitement. * Tests unitaires de l’inférence sur des images statiques (7 classes : colère, dégoût, peur, joie, neutralité, tristesse, surprise).

1.3 Intégration de CameraX

La capture vidéo en temps réel a été configurée via CameraX : * Implémentation du CameraProvider et configuration du Preview Use Case. * Mise en place du ImageAnalysis Use Case pour le traitement frame-by-frame. * Rotation et redimensionnement automatique des frames vers la résolution d’entrée du modèle (224×224 px). * Affichage du flux caméra dans un PreviewView intégré à Jetpack Compose via AndroidView.

1.4 Interface utilisateur de base

Un premier écran fonctionnel a été développé : * Écran principal avec PreviewView et overlay de résultats en temps réel. * Affichage du label d’émotion détectée et du score de confiance associé. * Bouton de basculement caméra avant / arrière. * Indicateur visuel de chargement du modèle et de l’état de la caméra.
